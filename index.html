字母替换解码与汉字的关系

但该方法对于中文这类非字母语言一直没有用武之地。 一些采用解码技术的中文的处理也都将单个汉字作为一个不再分割的单元进行处理。进而就会遇到问题空间太大的问题。

  • 一般字母语言的字母仅几十个；

  • 将单个汉字视为一个替换最小单位的情况下，一个汉字的替换可能空间在数千至数万的范围。

汉字的特点

  • 实际汉字并不如字母一般不可分割。

  • 汉字本身的部件有表音或表义的功能。 将其视为一个黑盒，将白白浪费许多有用的信息或线索。

  • 粗略的看其是有一笔一划组成的。

  • 细分的话，有多种划分方式。如，按笔画划分，按部首划分。

  • 但无论采用哪种方式划分，我们都可以认为汉字就是这些划分的一维编码。 为什么可以这么说呢？因为，在康熙字典中已明确给出了大部分汉字的笔顺

  • 字型信息 ＋ 笔顺， 汉字这一二维的文字就可以被转为一维编码。

字型信息编码 Stoke Feature Coding

我们将笔画用字母替代，并按汉字的笔顺编码就可以得到一维的汉字了。 如： “永”字八笔。

我们定义：最小单位为笔画。

如此，字母替换解码的方法将也能够在中文中使用。

统计方法与汉字部件信息

  • 汉字的部首等部件包含有意义或发音等信息。

  • 这些在传统的表示系统中需要通过附加的规则来进行表示，一些特例情况较难处理。

  • 将汉字转为一维表示后，我们可以通过统计的方式来这些相关的信息，特例将会依据前后的字母组合来滤除。

已有算法的应用

  • 基于汉字一维编码的语料的传统字母替换解码算法实践，减少替换问题空间。

  • 汉字是否将不再是一类特例来处理，所有应用于字母语言的技术都可应用于汉语这类都书写系统。

一维编码与Deep Learning

  • 在使用deep learning 时中文语料中的每一个字都是独立的黑盒。

  • 但如果我们使用汉字一维编码，我们可有一个新的层来统计部件（如笔画）之间的组合。

  • 一些表音的组合，或是偏旁信息便可被统计的到。作为下一个隐层的输入。

  • 带有更细粒度的隐层，是否更有助于提高算法的效果。
